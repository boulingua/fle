## 2. Mapping table — Quarto → Hugo

| Quarto construct | Hugo replacement | Notes |
|---|---|---|
| `_quarto.yml` site config | `hugo.toml` (TOML) | Match reference repo structure. |
| `format.html.theme: [flatly, …]` | Coder theme via Hugo Modules + `assets/css/custom.css` | Port SCSS overrides into Coder's customisable variables; preserve light/dark. |
| Navbar `website.navbar.left/right` | `[[menu.main]]` / `[[menu.footer]]` blocks | Preserve full structure incl. submenus. |
| Footer `website.page-footer` | `layouts/_partials/footer.html` (override Coder's) | Verbatim text + links. |
| `.qmd` body markdown | `.md` body (Goldmark, `unsafe = true`) | Verbatim copy. |
| `::: {.callout-note}` etc. | Custom shortcode `{{< callout type="note" >}}…{{< /callout >}}` (`layouts/shortcodes/callout.html`) | Render to same Bootstrap-callout HTML; preserve types `note`, `tip`, `warning`, `important`, `caution`. |
| `::: {.hero-kicker}`, `.lead`, `.card-grid`, `.card`, `.cefr-badge` | Plain `<div class="…">` via raw HTML (Goldmark `unsafe`) **or** thin shortcodes | Preserve class names; port matching CSS. |
| `[Text]{.cefr-badge.a1}` (Pandoc bracketed span) | Raw HTML `<span class="cefr-badge a1">Text</span>` | Goldmark does not parse Pandoc bracketed spans; rewrite at migration time. |
| Cross-refs `[Text](page.qmd)` | `[Text]({{< ref "page.md" >}})` or relative `[Text](../page/)` | Use `ref` shortcode for resolved-at-build links. |
| Images `![](path)` | Same; assets moved to `assets/` (processed) or `static/` (raw). | |
| `format: pdf` exam files | Out of Hugo's scope. **Keep building them via the existing Quarto-PDF pipeline** as a separate CI step that emits PDFs into `static/downloads/<track>/<unit>_exam.pdf`. Hugo then serves them as static assets. | This is the cleanest split: Hugo for HTML, Quarto-LaTeX for the 156 exam PDFs. Confirm this approach before Phase 2. |
| `_scripts/vgwort.lua` filter | **Discarded** (it is a no-op today). | |
| Plausible script tag | `layouts/_partials/head/plausible.html` (mirrors reference repo's `extensions.html`) | Verbatim copy of the snippet, only `data-domain` stays as-is. |
| VG Wort `::: {.vgwort-pixel}` body block | `layouts/_partials/vgwort.html` partial reading `Params.vgwort_pixel` (token) and emitting the same `<img>`; injected at end of `single.html`. | Migrator extracts the 32-hex token from each `.qmd` and writes it as `vgwort_pixel: <token>` in the new `.md` frontmatter. The body block is deleted. CI verifier confirms 1:1 token mapping. |

## 4. Risk list / open questions

1. **Reveal.js decks.** The README and `index.qmd` advertise Reveal.js slides per unit, but no `format: revealjs` was found in unit frontmatter. The slide generation must live somewhere I have not located yet — possibly an unrun phase. **Decision needed:** is slide rendering currently active? If yes, where? If no, is the new Hugo site expected to produce slides at all?
2. **Exam PDFs.** 156 `_exam.qmd` files render to PDF via LaTeX (`_includes/_exam.tex`). Hugo cannot do this. **Recommended split:** keep a slimmed-down Quarto-PDF step in CI that produces only `*_exam.pdf` into `static/downloads/`, and let Hugo serve them as static files. Confirm before Phase 2.
3. **Phase 3 placeholder materials clash with existing real worksheet placeholders.** `_scripts/make_placeholder_worksheets.py` already produces worksheet PDFs in CI. The migration prompt instructs me to create dummy `.pptx` + `.pdf` per article in `static/materials/`. **Decision:** do new placeholders supersede the existing worksheet pipeline, or do we wire the existing worksheet outputs into the Materials hub and only generate `.pptx` placeholders fresh? Strong recommendation: reuse existing worksheets, generate `.pptx` placeholders fresh.
4. **VG Wort pixels on listing pages.** Confirmed in prompt (no pixels on Materials hub). Will not pixel: `/`, `/materiel/*`, track index/uebersicht pages? Track index/uebersicht **already have pixels today** (e.g. `track_e_kl06/uebersicht.qmd` has `1be38254…`). They are revenue-bearing. **They keep their pixels.** Only the *new* hub pages (`/materiel/presentations/`, `/materiel/fiches/`) go un-pixeled.
5. **`docs/` directory is the current Quarto output dir.** It is also where the reference Hugo site would write to. Must verify `docs/` is not gitignored (it appears tracked) before deletion in Phase 4. GitHub Pages may currently serve from `docs/` on `main` rather than from a `gh-pages` branch — must check repo Pages settings or workflow. The current `publish.yml` actually uses `actions/upload-pages-artifact` against `docs/` then `deploy-pages`, so it is **artifact-based deployment**, not branch-based. The new Hugo workflow should match (artifact-based, building to `public/` by default).
6. **Content not pixeled today (4 files of 353).** Need explicit list confirmed in Phase 2 manifest before assuming legal pages are intentionally excluded.
7. **Lua filter on `_quarto.yml` line 86–87 is a no-op.** I will not port it, but want to flag in case the user expected it to be doing something.
8. **Pandoc bracketed spans `[X]{.cefr-badge.a1}`** appear in track index pages. Goldmark will not parse these — they must be rewritten to raw HTML at migration time. There may be other Pandoc-only syntax (e.g. `{#sec-…}` heading IDs) that needs a sweep.
